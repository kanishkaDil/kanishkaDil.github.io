02 / WHAT ───── What is Jenkins?
Jenkins is an open-source automation server that helps teams build, test, and deploy software continuously, reliably, and without manual intervention.
🖥️
Automation Server
Runs your build & test jobs automatically on every code change — no manual trigger needed.
free & open source · Java-based · self-hosted
📄
Pipeline as Code
Define your entire workflow in a Jenkinsfile — version-controlled, reviewable, and repeatable.
Groovy DSL · declarative syntax
🔌
Plugin Ecosystem
1,800+ plugins to integrate with Git, Docker, Kubernetes, Slack, AWS, Jira, and virtually everything else.
Git · Docker · K8s · Slack · AWS
⭐ 23k+ GitHub stars 🏢 Used by Netflix, LinkedIn, NASA 📅 Released 2011 (Hudson fork) ☕ Runs on JVM
03 / WHY ───── Why Jenkins?
✕ Without Jenkins
😩Manual builds — someone runs ./build.sh by hand on every merge
🤷"Works on my machine" — environment drift between devs
🕑Tests run only before major releases, bugs pile up
📦Deployments are manual, error-prone, and stressful
📉No audit trail — who deployed what and when?
🧩Onboarding takes days: "ask John how to release"
→
✓ With Jenkins
⚡Every git push triggers a build automatically via webhook
🐳Consistent Docker-based agents — same env every time
✅Tests run on every commit, failures reported in minutes
🚀One-click or fully automatic deployments to any environment
📋Full build history, logs, and stage-by-stage visibility
📖Jenkinsfile IS the runbook — self-documenting process

05 / JENKINSFILE ───── Components
Jenkinsfile 
pipeline {

  agent any          // run on any available agent

  environment {
    APP_NAME = 'my-service'
    DEPLOY_ENV = 'production'
  }

  stages {

    stage('Build') {
      steps {
        sh './gradlew build'
        archiveArtifacts 'build/libs/*.jar'
      }
    }

    stage('Test') {
      steps {
        sh './gradlew test'
        junit 'build/test-results/**/*.xml'
      }
    }

    stage('Deploy') {
      when { branch 'main' }
      steps {
        sh './deploy.sh $DEPLOY_ENV'
      }
    }

  }

  post {
    always  { cleanWs() }
    success { slackSend '✅ Build OK' }
    failure { slackSend '❌ Build FAILED' }
  }
}
pipeline { }
The root block. Everything lives inside this. Marks the file as a Declarative Pipeline.
agent
Where to run. any = any agent. Use docker, label, or kubernetes for specifics.
environment { }
Define env vars accessible in all stages. Can also inject from secrets managers.
stage('Name') { }
Logical grouping shown in the Jenkins UI. Makes progress visible. Each stage runs sequentially (or parallel with parallel).
steps { }
The actual commands: sh, bat, docker, git, withCredentials, custom steps.
post { }
Runs after all stages. Conditions: always, success, failure, unstable, cleanup.
when { }
Conditionally run a stage. Filter by branch, environment, expression, or tag.
